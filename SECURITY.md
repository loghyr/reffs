# Security Policy

## Supported Versions

The following versions of Reffs are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within Reffs, please send an e-mail to
Tom Haynes at loghyr@gmail.com. All security vulnerabilities will be promptly addressed.

Please include the following in your report:

- Type of issue (e.g., buffer overflow, use-after-free, etc.)
- Location of the vulnerability (file, function, etc.)
- Reproduction steps or a proof-of-concept (PoC)
- Potential impact

We ask that you do not report security vulnerabilities through public GitHub issues.
Instead, use the e-mail address above to allow us to address the issue before it
is made public.
